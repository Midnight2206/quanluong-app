Notes:
- This structure is `module-based`, not `layer-based`.
- Prefer flat files inside each domain module for normal-sized modules.
- Only create nested folders inside a module when the module becomes large enough to need subdomain separation.
- Shared infrastructure stays outside `modules/`, while business code stays inside `modules/<domain>/`.
