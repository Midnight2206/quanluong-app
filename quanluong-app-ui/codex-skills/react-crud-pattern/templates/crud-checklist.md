# CRUD Checklist

Su dung checklist nay khi scaffold hoac review mot CRUD feature.

1. Does the feature have a clear list screen?
2. Are filters, search, and pagination kept outside row-level components?
3. Are loading, empty, and error states handled explicitly?
4. Does create or update use a shared form pattern?
5. Is delete protected by confirmation UI?
6. Are mutation results surfaced through consistent feedback?
7. Is list data invalidated or refreshed after successful mutations?
8. Are reusable parts extracted only after real reuse appears?
