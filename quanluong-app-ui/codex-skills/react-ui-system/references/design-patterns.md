# Design Patterns

UI system nen co nhung pattern lap lai de user de hoc va de phat trien.

## Common Screen Patterns

- page header with title, context text, and primary actions
- filter toolbar above a list or table
- card-based detail sections
- modal confirmation for destructive actions
- form pages with grouped sections and sticky actions when needed

## Common Shared Components

- page header
- section card
- empty state card
- toolbar row
- status badge
- info list or key-value block

## Guardrails

- Do not invent a new screen pattern for every feature.
- Keep repeated page scaffolding in shared composed components.
- Use the same action placement pattern across similar screens.
