# Component Patterns

Khong phai component nao cung responsive theo cung mot cach.

## Tables

- prefer horizontal scroll only as a fallback
- consider card or stacked item layout on very small screens
- move secondary metadata below primary labels on mobile

## Forms

- stack fields vertically on mobile
- collapse side-by-side fields unless width truly allows it
- keep submit and cancel actions reachable without crowding

## Dashboards

- use fewer columns on smaller screens
- preserve metric hierarchy before preserving desktop symmetry

## Action Bars

- let actions wrap
- prioritize the main action
- hide non-critical controls behind menus when needed
